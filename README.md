# 一掷千金 · 红运临门

一个纯前端、可离线运行的"开红包版 Deal or No Deal"网页小游戏。24 个红包，一位神秘银行家，一路淘汰到底看看你能不能扛住诱惑。专为手机浏览器优化，适合过年过节、公司年会、家庭聚会现场投屏或传手机玩。

## 玩法说明

1. **选幸运红包**：24 个红包里先选 1 个作为自己的"幸运红包"，先不打开，留到最后。
2. **分轮淘汰**：接下来分好几轮，每轮要点开若干个（不是幸运红包的）红包，让它们"出局"——淘汰掉的金额会从奖金榜上划掉。
3. **银行家出价**：每淘汰完一轮，银行家都会来电，根据剩余红包的平均值出一个价格，问你「成交」还是「不成交」：
   - **成交**：直接拿走银行家出的这笔钱，游戏结束（幸运红包里原本是多少钱也会揭晓）。
   - **不成交**：不要这笔钱，继续淘汰下一轮。
4. **终局抉择**：淘汰到只剩 2 个红包（你的幸运红包 + 最后 1 个）时，银行家出完最后一次价后，你可以选择：
   - 坚持打开自己一开始选的幸运红包，或者
   - 换成最后剩下的那一个。
5. 最终金额结算，彩纸礼花走一波 🎉。

## 功能特色

- 🧧 24 红包 · 7 轮银行家出价 · 完整的终局交换机制
- 📞 银行家来电动画、报价弹窗、终局抉择弹窗，带音效和震动反馈（支持震动的设备）
- 🏆 奖金榜实时划掉已淘汰金额，深红金边高对比配色，一眼看清剩余奖金分布
- 📢 每轮开始时屏幕正中央会弹出醒目的"请淘汰 N 个红包"公告，不容易错过
- ⚠️ 红包淘汰揭晓画面明确标注「已淘汰」「原本价值」，避免误以为是自己中奖
- 🎵 背景音乐 + 选中/开红包/成交/不成交/来电/烟花等音效，来电时自动压低背景音乐
- 📱 已针对中低端手机做过渲染性能优化（去除高开销的 `backdrop-filter` 模糊、限制彩纸粒子数量等），开红包时的特效卡顿问题已改善
- 🔧 数字配置（奖金、轮次、银行家出价比例）已拆分到独立的 `config.js`，改数字不用碰主文件

## 怎么运行

不需要安装任何东西，也不需要联网（字体图标除外，见下方注意事项）：

1. 把整个文件夹解压到同一个目录下（`index.html`、`config.js`、`audio/` 必须在同一层级，路径关系不能变）。
2. 双击 `index.html`，用浏览器直接打开即可开始游戏。
3. 如果需要投屏到电视或者分享给朋友玩，把整个文件夹上传到任意静态网页托管服务（比如 GitHub Pages、Vercel、Netlify）即可，不需要后端。

> 手机端建议用 Chrome / Safari 最新版本打开，体验和性能优化效果最好。

## 怎么修改奖金数字

打开 **`config.js`** 这一个文件就够了，不需要动 `index.html`：

```js
const CONFIG = {
  currency: 'RM',                                        // 货币符号，改成 '¥'、'$' 都行
  prizes: [0.10, 0.50, 1, 5, ... , 1000],                // 24 个奖金金额（必须刚好24个）
  roundPlan: [6, 5, 4, 3, 2, 1, 1],                       // 每一轮要淘汰的红包数量，总和必须=22
  offerPercents: [0.15, 0.30, 0.45, 0.55, 0.65, 0.75, 0.90] // 每一轮银行家出价占剩余平均值的比例
};
```

规则速记：

| 字段 | 要求 |
|---|---|
| `prizes` | 必须是 **24 个**数字，对应 24 个红包 |
| `roundPlan` | 各项加起来必须等于 **22**（24 个红包 − 1 个幸运红包 − 1 个留到终局） |
| `offerPercents` | 项数必须和 `roundPlan` 一致，每项是 0~1 之间的比例 |

改完保存，刷新页面就会生效，不需要重新下载或安装任何东西。

## 怎么替换音效/背景音乐

把 `audio/` 文件夹里对应的 mp3 文件换成你自己的文件（**文件名保持不变**）即可，`index.html` 里通过固定文件名引用，不需要改代码。

## 技术说明
